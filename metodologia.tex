\section{Metodologia}\label{sec:metodologia}
Para haver solidez estatística e também para uma boa representatividade dos variados domínios
existentes em aplicações reais, foram selecionadas as cem bases de dados mais diversas
do projeto \red{ucipp <- refenciar como?}.
O critério de diversidade envolveu manter apenas uma dentre cada grupo de bases de um mesmo domínio.
Com exceção de algumas que tivessem características (atributos, classes e número de exemplos)
suficientemente diferentes ou cuja acurácia passiva fosse suficientemente distinta para pelo
menos três classificadores.

As \elms e as estratégias baseadas na métrica de distância de Mahalanobis precisaram ser transformadas com a
substituição de atributos nominais por numéricos e pela aplicação do \engl{filtro de padronização}{z-score}
para possibilitar os cálculos e para evitar matrizes mal condicionadas - respectivamente.
Os demais algoritmos de aprendizado fizeram eventual uso de seus próprios filtros internos
providos pela ferramenta Weka \citep{journals/sigkdd/HallFHPRW09}.

\subsection{Cenário}\label{sec:cenario}

kappa:\citep{conf/pkdd/Shah11}
accbal: \citep{journals/bmcbi/MassoV10}

O cenário detalhado adotado neste trabalho é baseado em \pool;
de consulta por classe monorrótulo; com amostragem um a um de uma distribuição estacionária com
custos uniformes; com oráculo único e consistente,
sendo um rótulo inicial por classe simuladamente obtido via
\engl{aprendizado guiado}{guided learning} \citep{conf/kdd/AttenbergP10};
com atributos sem valores faltantes;
e, sem redução do tamanho das bases de dados originais\footnote{Exceto para a estratégia de \eer,
para ser computacionalmente tratável.}.
